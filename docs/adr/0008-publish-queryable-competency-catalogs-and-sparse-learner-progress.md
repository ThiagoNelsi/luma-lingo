# ADR 0008: Publish queryable competency catalogs and sparse learner progress

Competency catalogs will be authored as versioned JSON files, then published into queryable runtime storage for planning, lesson generation, diagnostics, and progress tracking. Runtime catalog data will keep competencies, prerequisite relationships, and goal priorities as first-class queryable concepts; richer generation context such as examples, forms, tags, and common learner errors may remain embedded until query needs justify further normalization.

Learner progress will be stored sparsely per `LearningTrack`: the system records competency state only after diagnostic, lesson, review, or activity evidence exists, treating missing state as unknown rather than not mastered. The planner will choose the next module through `Module candidate score`: prerequisite readiness gates or strongly penalizes candidates first, then weighted criteria such as goal priority, competency gap, confidence need, review urgency, lesson emphasis fit, and recent focus determine the best next module objective. Early lessons therefore teach while also collecting evidence that sharpens the learner's competency profile.
